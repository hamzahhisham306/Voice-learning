المشاكل التي واجهت مشروع Converso وكيف تمّ حلّها

شرح مبسّط لكل مشكلة ظهرت أثناء إعداد التطبيق: ما هي، وما سببها الحقيقي، وكيف تمّ إصلاحها خطوة بخطوة.

المشروع: Converso (Next.js) المصادقة: Clerk قاعدة البيانات: Supabase يوليو ٢٠٢٦

المشاكل التي تمّ إصلاحها

01

إعداد مصادقة Clerk (كان يعمل بوضع «بدون مفاتيح»)

تمّ الإصلاح
المشكلة

كان التطبيق يعمل في «الوضع بدون مفاتيح» (keyless mode) وملف .env فارغاً، أي أنه لم يكن مرتبطاً بحساب Clerk حقيقي، فلا يمكن تسجيل الدخول فعلياً.

السبب

لم تكن مفاتيح Clerk (المفتاح العام والسرّي) مضبوطة، وكان المشروع يعتمد على مفاتيح تجريبية مؤقتة أنشأها Clerk تلقائياً.

الحل
  1. تثبيت أداة Clerk CLI عبر npm install -g clerk.
  2. تسجيل الدخول إلى الحساب عبر clerk auth login.
  3. تشغيل clerk init الذي ربط المشروع بتطبيق Clerk حقيقي باسم jsm_saas_app، وكتب المفاتيح في .env، وأنشأ صفحة التسجيل sign-up.
ملفات: .envmiddleware.tsapp/layout.tsx
02

تعطُّل الصفحة الرئيسية: «supabaseUrl is required»

تمّ الإصلاح
المشكلة

كانت الصفحة الرئيسية تتعطّل فور فتح التطبيق برسالة الخطأ:

Error: supabaseUrl is required.
  at createSupabaseClient (lib/supabase.ts:5)
السبب

متغيّرات بيئة Supabase لم تكن موجودة أصلاً. والأهم: الكود في lib/supabase.ts كان يقرأ اسم متغيّر خاطئ (NEXT_PUBLIC_SUPABASE_ANON_KEY)، بينما قيمة المفتاح لديك موجودة تحت اسم مختلف (NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY).

الحل
  1. إنشاء ملف .env.local يحتوي رابط المشروع والمفتاح العام (Publishable Key).
  2. تعديل lib/supabase.ts ليقرأ الاسم الصحيح NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY.
ملفات: .env.locallib/supabase.ts
03

خطأ بعد تسجيل الدخول: «No suitable key or wrong key type»

تمّ الإصلاح
المشكلة

عند تسجيل الدخول، كانت صفحة الرفقاء والصفحة الرئيسية تتعطّلان بالخطأ التالي — بينما كان كل شيء يعمل عند عدم تسجيل الدخول:

Error: No suitable key or wrong key type
  at getAllCompanions (lib/actions/companion.actions.ts:39)
السبب

عند تسجيل الدخول، يرسل التطبيق «توكن» Clerk إلى Supabase للتحقق. لكن مشروع Supabase كان يثق بـنطاق Clerk قديم بينما أصبح التطبيق يستخدم نطاقاً جديداً، فلم تجد Supabase مفتاحاً للتحقق من التوكن.

# النطاق الذي كانت تثق به Supabase (قديم)
accepted-buffalo-35.clerk.accounts.dev
# النطاق الذي يستخدمه التطبيق فعلياً (جديد)
sure-piranha-43.clerk.accounts.dev
الحل
  1. في لوحة تحكم Clerk: تفعيل تكامل Supabase (يضيف «مطالبة الدور» role claim إلى التوكن)، ونسخ نطاق Clerk الصحيح.
  2. في لوحة تحكم Supabase ← Authentication ← Third-Party Auth: إضافة مزوّد Clerk بالنطاق الصحيح sure-piranha-43.clerk.accounts.dev.

النتيجة بعد الإصلاح — تحوّل الطلب من فشل إلى نجاح:

GET /companions 500  ← قبل
GET /companions 200  ← بعد
أُنجز في: Clerk DashboardSupabase Dashboard
04

بطء التنقّل بين الصفحات (تجربة المستخدم)

تمّ الإصلاح
المشكلة

عند الضغط على أي رابط أو زر للانتقال، كان هناك تأخير بحدود ثانية قبل ظهور الصفحة الجديدة — وكأن الضغطة لم تُسجَّل.

الأسباب
  • لا توجد ملفات loading.tsx، فتبقى الصفحة القديمة «مجمّدة» حتى ينتهي الخادم من جلب البيانات (لا يوجد مؤشّر تحميل فوري).
  • جلب البيانات كان متسلسلاً (واحداً تلو الآخر) بدل التوازي.
  • رابط «My Journey» كان خاطئاً (my-journey بدون /)، فيؤدي أحياناً لمسار خاطئ.
  • تسريب في مؤقّت البحث (debounce) يُطلق طلب تنقّل جديد مع كل ضغطة زر.
  • زر «Build a New Companion» كان <button> يحيط بـ<Link>، ما يمنع التحميل المسبق (prefetch).
الحلول
  • إضافة «هياكل تحميل» (loading skeletons) لكل صفحة ← انتقال فوري مع بقاء القائمة العلوية ثابتة.
  • جلب البيانات بالتوازي عبر Promise.all.
  • تصحيح رابط «My Journey» إلى /my-journey.
  • إصلاح البحث: تنظيف المؤقّت + useTransition + مؤشّر تحميل صغير.
  • تحويل زر الإنشاء إلى رابط قابل للتحميل المسبق، وإضافة حالة «جارٍ الإنشاء…» للزر.
ملفات: app/loading.tsxapp/companions/loading.tsxNavItems.tsxSearchInput.tsxCTA.tsx

نقاط تحتاج انتباهك

⚠ مشكلتان متبقّيتان (منفصلتان عمّا سبق)

١. خطأ مؤقّت «JWT not yet valid»

سببه فرق بسيط في توقيت الساعة بين Clerk وSupabase (clock skew). يظهر أحياناً ويختفي عند إعادة تحميل الصفحة. إن تكرّر كثيراً فالحل هو مزامنة ساعة الجهاز.

٢. جدول «bookmarks» مفقود في قاعدة البيانات

صفحة «My Journey» تتعطّل لأن قاعدة بيانات Supabase تنقصها جداول (bookmarks وربما session_history). جدول companions موجود ويعمل، لكن يلزم إنشاء الجداول الناقصة لتعمل صفحة الرحلة والإشارات المرجعية.

الخلاصة — حالة التطبيق الآن

✓ مصادقة Clerk تعمل (تسجيل الدخول والتسجيل).
✓ قراءة بيانات Supabase تعمل بعد تسجيل الدخول.
✓ التنقّل بين الصفحات أصبح سلساً وفورياً.
! يتبقّى: إنشاء الجداول الناقصة في Supabase.
